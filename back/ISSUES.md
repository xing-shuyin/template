# Back 项目问题分析与优先级排序

## P0 - 必须立即修复（安全漏洞）

### 1. 密码哈希算法弱
- **问题**：使用 `pbkdf2_sha256`，而非更安全的 bcrypt/argon2
- **影响**：密码泄露风险高，pbkdf2 已被证明抗暴力破解能力不足
- **位置**：`security.py:23`
- **修复**：改用 `bcrypt` 或 `argon2`

### 2. 无时序攻击防护
- **问题**：认证时未对不存在的用户使用虚拟哈希，攻击者可通过响应时间判断用户是否存在
- **影响**：可枚举有效用户名
- **位置**：`security.py:40-42`
- **修复**：对不存在的用户也执行一次虚拟哈希比较

### 3. 缺少密钥验证
- **问题**：`SECRET_KEY` 默认值是 `secrets.token_urlsafe(32)`，生产环境可能使用随机密钥
- **影响**：重启后所有 token 失效；或密钥强度不够
- **位置**：`settings.py:30`
- **修复**：生产环境强制要求通过环境变量设置 SECRET_KEY，否则启动报错

## P1 - 尽快修复（影响稳定性/可维护性）

### 4. 无 Alembic 迁移
- **问题**：虽有 alembic 目录，但实际使用 `SQLModel.metadata.create_all()`
- **影响**：生产环境无法安全升级数据库结构，无法回滚
- **位置**：`db.py:85`
- **修复**：使用 alembic 生成并管理迁移脚本

### 5. 完全没有测试
- **问题**：零测试覆盖，无 tests 目录
- **影响**：无法保证代码正确性，重构风险极高
- **修复**：添加 pytest + httpx 异步测试，覆盖核心业务逻辑

### 6. 混用同步/异步引擎
- **问题**：同时存在 `engine`（同步）和 `async_engine`（异步），使用场景不清晰
- **影响**：连接池资源浪费，可能导致连接耗尽
- **位置**：`db.py:13-20`
- **修复**：统一使用异步引擎，仅在 alembic 迁移等必要场景使用同步引擎

### 7. 敏感端点缺少限流
- **问题**：仅登录有限流，密码重置、注册等敏感操作无保护
- **影响**：暴力破解、恶意注册风险
- **修复**：对所有敏感端点添加 Redis 限流

## P2 - 计划修复（代码质量/架构）

### 8. 无 CRUD 层
- **问题**：数据库操作直接混在路由处理器中
- **影响**：代码重复，难以测试和维护
- **修复**：引入 `crud/` 层，分离业务逻辑与路由

### 9. 扁平化结构
- **问题**：所有代码在 `src/` 根目录，无子模块划分
- **影响**：项目扩大后难以导航和维护
- **修复**：按功能模块组织为 `app/api/`、`app/core/`、`app/models/` 等

### 10. 循环导入风险
- **问题**：`utils.py` 导入 `src.deps`，`deps.py` 又导入 `models.py`
- **影响**：可能导致运行时 ImportError
- **修复**：重构模块依赖关系，消除循环引用

### 11. 缺少级联删除
- **问题**：关系未配置 `ondelete="CASCADE"`
- **影响**：删除用户时相关数据（角色、部门关联等）残留，可能引发外键约束错误
- **修复**：在 Relationship 中配置级联删除

### 12. 密码重置流程不完善
- **问题**：直接更新密码，缺少完整的令牌验证流程
- **影响**：安全风险，可能被利用进行账户接管
- **修复**：实现带过期时间的一次性令牌验证

### 13. 无邮箱枚举防护
- **问题**：密码恢复时会暴露邮箱是否已注册
- **影响**：攻击者可枚举系统邮箱
- **修复**：无论邮箱是否存在，统一返回"已发送重置邮件"

### 14. 无刷新令牌轮换
- **问题**：基本的刷新机制，没有令牌轮换策略
- **影响**：刷新令牌泄露后无法自动失效
- **修复**：实现 refresh token rotation

## P3 - 有空再改（优化项）

### 15. 异步使用不一致
- **问题**：部分路由用同步 DB session，部分用异步
- **修复**：统一为异步

### 16. 类型提示不完整
- **问题**：整个项目类型标注不一致
- **修复**：添加 mypy 配置，逐步补全类型

### 17. 使用 print 语句
- **问题**：部分地方用 `print()` 而非 loguru
- **位置**：`models.py` 中的 `before_delete` 钩子
- **修复**：统一使用 loguru

### 18. 硬编码路径
- **问题**：媒体路径用 `os.path.join` 构造
- **位置**：`settings.py` 的 `MEDIA_PATH`
- **修复**：使用 `pathlib.Path`

### 19. 使用过时的包
- **问题**：使用 `passlib` 而非现代的 `pwdlib`
- **修复**：迁移到 `pwdlib`

### 20. 无代码质量工具
- **问题**：缺少 ruff、mypy 等 linting/formatting 配置
- **修复**：在 `pyproject.toml` 中添加 ruff/mypy 配置

### 21. 无 Docker 配置
- **问题**：缺少 Dockerfile 和 docker-compose
- **修复**：根据部署需求添加容器化配置

### 22. 无 Sentry 集成
- **问题**：缺少生产环境错误追踪和监控
- **修复**：集成 sentry-sdk

### 23. 无预启动脚本
- **问题**：缺少数据库连接检查和初始化脚本
- **修复**：添加 prestart.sh

### 24. 手动初始化
- **问题**：`init_db()` 在 `__main__` 块中运行，而非 FastAPI lifespan
- **位置**：`main.py:63-64`
- **修复**：移入 lifespan 的 startup 阶段

### 25. 响应模型不一致
- **问题**：部分路由返回字典而非类型化的 Pydantic 模型
- **修复**：统一使用 `response_model`

### 26. 无 OpenAPI 定制
- **问题**：缺少自定义路由 ID 生成
- **修复**：自定义 `generate_unique_id_function`

### 27. CORS 配置不够严格
- **问题**：CORS 来源验证较宽松
- **修复**：生产环境严格限制允许的 origins

### 28. 版本约束宽松
- **问题**：pyproject.toml 中依赖版本约束不够严格
- **修复**：锁定关键依赖版本
